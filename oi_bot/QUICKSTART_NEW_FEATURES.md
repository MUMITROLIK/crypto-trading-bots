# 🚀 Quick Start — Новые фичи OI Bot

## Что добавлено?

✅ **Order Book Analysis** — крупные стенки покупок/продаж  
✅ **Footprint** — дельта покупок/продаж по ценовым уровням  
✅ **Volume Profile** — зоны где цена тусовалась дольше всего  

---

## 📦 Установка (если ещё не установлено)

```bash
cd oi_bot
pip install -r requirements.txt
```

**Зависимости:**
- aiohttp >= 3.9
- websockets >= 12.0
- python-dotenv >= 1.0

---

## 🎯 Запуск

```bash
cd oi_bot
python main.py
```

**Что происходит:**
1. Загружаются символы с Binance (535+) и Bybit (400+)
2. Подключаются WebSocket потоки (цены, OI, торговые потоки)
3. Начинается накопление данных
4. Через 2-3 минуты начнут приходить сигналы

## 📊 Как работают новые фичи?

### 1. Order Book (стенки)

**Когда активируется:**
- При каждом сигнале агрегатора
- Запрашивает топ-20 уровней стакана
- Кэш 30 секунд (не спамим API)

**Что ищет:**
- Крупные ордера (в 3x больше среднего И >= $8K)
- Стенка продаж 0-4% выше цены → сопротивление → +1 балл SHORT
- Стенка покупок -4-0% ниже цены → поддержка → +1 балл LONG

**Пример в сообщении:**
```
🧱 Стенка продаж: 67,850 (+2.3%, $45K)
🛡 Стенка покупок: 66,100 (-2.0%, $38K)
```

---

### 2. Footprint (дельта по уровням)

**Когда активируется:**
- Автоматически при CVD подписке (OI >= 3%)
- Группирует сделки по ценовым уровням (~0.2% разрешение)
- Считает дельту покупок/продаж

**Что показывает:**
- Footprint > $1K → покупатели доминируют → +1 балл LONG
- Footprint < -$1K → продавцы доминируют → +1 балл SHORT

**Пример в сообщении:**
```
🟢 Footprint: покупатели $12.5K
🔴 Footprint: продавцы $8.3K
```

---

### 3. Volume Profile (зоны объёма)

**Когда активируется:**
- При каждом сигнале агрегатора
- Анализирует price_history (тысячи точек за 2 часа)
- Находит уровни где цена "тусовалась" дольше всего

**Что показывает:**
- 3 ключевых уровня поддержки/сопротивления
- Трейдер использует для выбора точек входа/выхода

**Пример в сообщении:**
```
📊 Volume зоны: 66,200 / 67,100 / 68,500
```

## 🎯 Скоринг (обновлённый)

### SHORT сигнал (макс 21 балл, порог 10)

**Базовые факторы (до 19 баллов):**
- Памп 1ч/20м: до 6 баллов
- OI в боковике после пампа: 4 балла
- L/S ratio > 63%: 3 балла
- Funding > 0.05%: 2 балла
- LONG ликвидации: 2 балла
- CVD < 0: 2 балла

**Новые факторы (+2 балла):**
- Footprint продавцы > $1K: +1 балл
- Стенка продаж 0-4%: +1 балл

---

### LONG сигнал (макс 16 баллов, порог 7)

**Базовые факторы (до 14 баллов):**
- OI +5% + рост цены: 3 балла
- SHORT ликвидации: 2 балла
- CVD > 0: 2 балла
- L/S ratio < 38%: 3 балла
- Цена 15м/1ч: до 2 баллов
- Funding умеренный: 1 балл
- Отскок от падения: 2 балла

**Новые факторы (+2 балла):**
- Footprint покупатели > $1K: +1 балл
- Стенка покупок -4-0%: +1 балл

## 🐛 Troubleshooting

### Footprint не отображается
**Причина:** CVD tracker не активен (OI < 3%)  
**Решение:** Подожди пока монета наберёт OI >= 3%

### Стенки не отображаются
**Причина:** Нет крупных стенок близко к цене  
**Решение:** Это нормально, не на каждой монете есть стенки

### Volume зоны пустые
**Причина:** Недостаточно данных (< 10 точек в price_history)  
**Решение:** Подожди 2-3 минуты после старта

### Ошибка 429 Too Many Requests
**Причина:** Превышен лимит API  
**Решение:** Увеличь `_CACHE_TTL` в orderbook.py с 30 до 60 секунд
